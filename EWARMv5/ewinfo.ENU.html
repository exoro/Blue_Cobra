Example description

  
DESCRIPTION
===========
   This demo project shows how to use the IAR Embedded Workbench for ARM to
  develop code for the STM3210E-Eval board. It is a demonstration firmware
  which you can use to evaluate the capabilities of the microcontroller and
  the on-board peripherals.
   
COMPATIBILITY
=============
   The STM32F10E_EVAL_Demo project is compatible with STMicroelectronics  
  STM32F10E_EVAL board. By default, the project is configured to use the
  J-Link JTAG/SWD interface.
  
CONFIGURATION
=============

  You can find the board's user manual at http://www.st.com.
  
GETTING STARTED
===============

  1) Start the IAR Embedded Workbench for ARM.

  2) Select File->Open->Workspace...
     Open the following workspace:

     \arm\examples\ST\
     STM32F10x\STM32-Eval\STM3210E-EVAL_Demo
     \project\Demo\EWARMv5\STM3210E-EVAL_Demo.eww

  3) Run the program.

     The STM32F10E_EVAL_Demo is downloaded to the Embedded Flash memory
     on the evaluation board and executed.
     
     Note! The "Debug" configuration will not download the files used by the
     demonstration (wav file, product presentation, icons) that must be
     programmed into the NOR Flash memory (mounted on the STM3210E-EVAL board).
     Please refer to board's user manula how to program the 
     STM3210E-EVAL_Demo_NORFlash_V2.0.0.dfu image or use "Debug_NOR_update"
     configuration.
     
     Note! The "Debug_NOR_update" configuration will download the Demo and the
     files image STM3210E-EVAL_Demo_NORFlash_V2.0.0.bin. This however will
     take more time and we recomend you to use "Debug_NOR_update" configuration
     only when the icons and bitmaps files are missing from NOR memory.
